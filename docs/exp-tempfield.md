## 1. 目的

验证 §3.2 + §5.1.1 温度场的行为是否符合预期:

1. `T = tanh(w1·P + w2·R + w3·E)` 归约函数
2. Hysteresis 滞后带(0.6/0.4)+ 温变率限幅 ±0.2/tick
3. 采样稀疏 + 衰减 0.9/tick
4. dual gate:温度场 + AgentLoad 任一触发即降级

**M5.5 的实际目标**:**实现能跑**,不是拟合出真实参数。真实参数校准待 Paper 实装 + 真实玩家数据(M6+ 之后)。

---

## 2. 方法

### 2.1 数据生成(合成)

- **真值权重**:`TrueWeights = {W1=1.2, W2=0.8, W3=1.5}`(用来生成样本真值温度,sim 不知道)
- **抽样分布**:
  - `player_proximity` ~ Beta(2, 1) 偏 0(大多数 chunk 无人)
  - `redstone_activity` ~ Uniform(0, 0.8) 偏中低
  - `entity_density` ~ Beta(1.5, 1)·0.9 偏 0
- **观测噪声**:±5% 均匀噪声
- **样本数**:200 sim × 100 chunk × 20 tick = 400,000 样本

### 2.2 三数据集切分(60/20/20)

- train:sim 0-119(120 sim,240,000 样本)— 拟合用
- val:sim 120-159(40 sim,80,000 样本)— 选最优参数
- test:sim 160-199(40 sim,80,000 样本)— **只看一次**

### 2.3 拟合方法

- 简单网格搜索:w1/w2/w3 各 0.5..2.0,步长 0.1
- 评估:val loss 最低 = 最优
- **不**用 test 调参

### 2.4 过拟合判定(§10.5 三指标)

1. **Train/Val gap** < 5% 安全,5-15% 警告,> 15% 过拟合
2. **Test/Val gap** < 10% 泛化 OK,> 10% 调参偷看 val
3. **参数稳定性**:训练集分 5 份独立拟合,差异 < 20% 稳定

## 4. 已知局限

### 4.1 合成数据(主因)

- 真值权重 `TrueWeights` 是 sim **自己知道**的,网格搜索只能找到它
- 真实 MC 玩家行为模式 / 红石频率 / 实体密度分布**不**代表这个合成分布
- 玩家移动模式是均匀随机,而真实玩家行为**高度偏斜**(长时间停留 / 快速跑过 / 转圈挖矿)
- **结论**:这个实验**没有验证任何关于真实 MC 的事**

### 4.2 参数稳定性 0 不可信

- 5 份独立训练**都收敛到同一解**——因为合成数据无随机性,无差异
- 真实数据会显示**非零** stability range(分 5 份训练子集,fit 结果会有差异)
- stability range = 0 在真实数据上是**过拟合的早期信号**(参数对任何训练子集都不敏感 → 唯一解)

### 4.3 噪声小

- ±5% 均匀噪声对真值 1.0 量级的权重**几乎不产生差异**
- 真实测量噪声更大(玩家位置噪声 / 实体计数 race / 红石 tick 采样误差)
- 真实数据下 train/val gap 会显著高于 0.66%
