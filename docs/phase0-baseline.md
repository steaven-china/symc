## 3. 方法(过拟合判定)

### 3.1 什么叫"过拟合"(symc 语境)

symc 实验是**校准参数**(w₁/w₂/w₃、hysteresis 带宽、缓冲带宽度 N、Load 阈值等),目标是 sim 跑出的"最优参数"在 **Paper 实装 + 真实玩家**时仍有效。**过拟合 = 在 sim 内样本上调到最优,但换真实场景/换玩家行为模式就拉垮**。

### 3.2 判定方法:数据分割 + 留出集

**核心思路**:把 sim 的输入数据切成三份,只用其中一份调参。

| 数据集 | 用途 | 占比 | 不可见 |
|---|---|---|---|
| **训练集**(train) | 调参数(w₁/w₂/w₃ 等) | 60% | 调参时只看这个 |
| **验证集**(val) | 选"哪个参数组合最好" | 20% | 调参时只看不参与计算梯度 |
| **测试集**(test) | 最终评估"这个最好组合在新数据上表现如何" | 20% | **整个调参过程完全不看** |

**关键规则**:**测试集只在最后用一次**。如果用测试集反复调,测试集就退化成训练集,过拟合检测失效。

### 3.3 判定指标(三个并用)

**指标 1:Train/Val gap(过拟合早期信号)**
```
gap = (val_score - train_score) / train_score
```
- gap < 5% → 未过拟合
- gap 5% ~ 15% → 警告,可能过拟合
- gap > 15% → 已过拟合,丢弃当前参数组合重新调

**指标 2:Test/Val gap(泛化能力)**
```
final_gap = (test_score - val_score) / val_score
```
- final_gap < 10% → 泛化 OK,采纳 val 选的参数
- final_gap > 10% → 调参过程中 val 被偷看过,丢弃整个调参过程,重新划分数据再调

**指标 3:参数稳定性(参数对训练子集的敏感度)**
- 把训练集随机分 5 份,每份独立跑调参
- 5 份得到的"最优 w₁/w₂/w₃"如果差异 < 20% → 稳定
- 差异 > 20% → 参数对训练数据过敏感,实际使用中可能不稳定

### 3.4 symc 特殊点(跟标准 ML 过拟合的差异)

- **不是神经网络调参**,是 sim 调结构化参数(权重向量 + 阈值),参数空间小(< 10 维)
- **sim 是确定的**(同输入同输出),不需要随机种子平均
- **真实场景 = Paper 实装 + 真实玩家**——sim 拟合再好,Paper 集成时可能差很多
- **过拟合主要风险**:sim 抽样的玩家行为模式 / 红石频率 / 实体密度分布**不**代表真实 MC 世界

**额外检查**:
- 训练集抽样要**广**:多种玩家密度(空 / 稀疏 / 中 / 密 / 满)、多种红石频率(无 / 偶发 / 持续)、多种实体密度(空 / 中 / 满)
- val/test 抽样**不能**跟 train 同一批 sim——要从**不同的种子**生成
- sim 的"玩家移动"模式要**不规则**——不能全是直线 / 全是随机,要有"在某个 chunk 站很久""快速跑过"等真实行为

### 3.5 不可过拟合兜底

如果 sim 跑出来参数过拟合严重,**保留**初值(w₁=w₂=w₃=1.0、hysteresis 0.6/0.4、缓冲带 N=2)作为**默认值**,sim 拟合的"最优"只作**参考**,不当硬性参数。原因:
- 真实 MC 玩家行为分布跟 sim 抽样**必然有偏差**
- 初值是按"无信息先验"假设三个因子等权,虽然粗糙但**不会系统性地过拟合**
- 拟合最优可能只对 sim 抽样过拟合,对真实场景**反向**
