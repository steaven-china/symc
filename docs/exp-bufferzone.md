# 实验报告 · P4 缓冲带 sim

> **实验 ID**:exp-bufferzone · 2026-06-20
> **状态**:✅ 跑通(synthetic data,真值 N=2 拟合出 N=2)
> **产物**:`results/bufferzone-fit.json`
> **代码**:`cmd/exp-bufferzone/main.go`

---

## 1. 目的

测"缓冲带宽度 N vs CompositeEvent 减少量 vs 算力税"——找最优 N。

仿 M5.5 架构:合成数据 + 三数据集切分 + 网格搜索拟合。

---

## 2. 方法

- 8×8 chunks 世界(64 chunks),4×4 region(4 region 总)
- 10 玩家随机游走 100 tick
- **简化模型**:玩家最终位置 → 距离 region 边界距离 → 跨区事件
- 缓冲带宽度 N:玩家距 region 边界 < N 时,**不算跨区事件**(邻居 region 已在缓冲带模拟)
- 算力税:缓冲带内每 chunk 2× 模拟(基础 1× + 邻居校验 1×)
- 变量:N ∈ {0, 1, 2, 3, 4}
- 数据集:200 sim,三切分(60/20/20)
- **真值 N=2**(假设最优,sim 拟合目标)

---

## 4. 关键发现

### 4.1 N=1 是临界点(性价比拐点)

- **N=0**:10 events(全跨区,纯 baseline)
- **N=1**:2.4 events(76% 减少,代价 +25% 算力)
- **N=2**:0 events(完全消除,代价 +50% 算力)
- **N=3+**:0 events(没额外收益,纯算力浪费)

**推荐**:**N=1** 是性价比最优——76% 减少 + 25% 算力税;N=2 在"必须零跨区事件"的场景下用。

### 4.2 算力税与 §11 风险表对应

§11 风险表(2026-06-20)已加"缓冲带算力税"条目:
- **N=0**:64 cost = 1.0× baseline
- **N=1**:80 cost = 1.25× baseline
- **N=2**:96 cost = 1.5× baseline
- **N=4**:128 cost = 2.0× baseline(对应 §5.2.1 "2× 算力"理论值)

数据印证 §5.2.1 / §11 的估计——**N=4 时算力真翻倍**,N=2 时 +50%。

### 4.3 拟合 trivial 但验证模型

- 跟 M5.5 一样:合成数据 + TrueWidth 已知 → 网格搜索找到同解
- **不**提供真实校准价值,但**模型正确性**已验证:
  - N=0 跑出 10 events(预期,无缓冲带)
  - N=2 跑出 0 events(预期,完全覆盖)
  - cost 跟 N 线性 +16 cost/N(预期,缓冲带宽度 16 chunk × 2 = 32)
  - fitted N=2 跟 true N=2 一致(模型逻辑对)

---

## 5. 局限

### 5.1 合成数据 + 简化模型

- 玩家是**最终位置**,不是轨迹(真实玩家跨区是渐进的)
- region 大小编码简化(4×4 固定)
- 玩家密度固定 10 个,实际 MC 玩家密度时空分布偏斜(中位数为 0,长尾)
- **不**模拟 MC 的红石/水流/实体跨区具体行为

### 5.2 拟合无意义但模型验证有

- TrueWidth 已知 → 网格搜找到同解
- 真实场景 N 未知,要靠 Profile 测"实际跨区事件率 vs 缓冲带 N"找拐点

## 6. 结论

**P4 跑通**。模型逻辑正确,数据印证 §5.2.1 / §11 缓冲带算力税估计:

| 决策维度 | N=1 | N=2 |
|---|---|---|
| CompositeEvent 减少 | 76% | 100% |
| 算力税 | +25% | +50% |
